B/S架构如何测试

发表于 2022-02-04 分类于 测试 阅读次数：

随着信息技术的发展，人们越来越习惯与在网上搜索信息来做出决定，因此网站已经是各类企业的必备之选。但是仅仅拥有一个网站还不够，还需要提供一个信息丰富、可访问和用户友好的网站。为了实现这样的功能，网站要经过良好的测试，这样的测试测试成为B/S架构测试即页面测试。

什么是页面测试？

页面测试是软件测试中的一种，用于测试网站或者web类应用程序是否存在一些问题。在上线之前要对基于页面的应用程序进行完整测试。

基于页面的系统在面向最终用户上线之前，需要进行端到端的全面的测试。

通过页面测试，可以确保基于页面的系统正常运行，且可以被用户接受，UI测试和功能测试是页面测试的首选。

常见的页面测试类型

  • 功能测试
  • 可用性测试
  • 接口测试
  • 兼容性测试
  • 性能测试
  • 安全测试
功能测试

测试页面中所有的链接，数据库连接，页面中用于提交或获取用户信息的表单，Cookie测试等

链接测试
  • 测试从所有页面到受测特定域的传出链接。
  • 测试所有内部链接。
  • 测试链接跳转到同一页面。
  • 测试链接用于从网页向管理员或其他用户发送电子邮件。
  • 测试以查看是否存在任何孤立页面。
  • 最后，链接检查包括，检查上述所有链接中的断开链接。
表单测试

表单在任何网站都是不可或缺的，一般用于接收来自用户信息。如何进行表单测试呢？

  • 首先，检查每个字段的所有验证。
  • 检查字段中的默认值。
  • 表单中对表单中字段的错误输入。
  • 用于创建表单的选项（如果有）、表单删除、查看或修改表单。
cookie测试

Cookie 是存储在用户计算机上的小文件。这些基本上用于维护会话 - 主要是登录会话。通过在浏览器选项中启用或禁用 Cookie 来测试应用程序。

在写入用户计算机之前，测试 Cookie 是否已加密。如果您正在测试会话 Cookie（即在会话结束后过期的 Cookie），请在会话结束后检查登录会话和用户统计信息。通过删除 Cookie 来检查对应用程序安全的影响。

HTML/CSS测试

如果您正在为搜索引擎优化您的网站，那么HTML / CSS验证是最重要的。主要验证网站的HTML语法错误。检查该网站是否可被不同的搜索引擎抓取。

数据库连接测试

数据一致性在 Web 应用程序中也非常重要。在编辑、删除、修改表单或执行任何与数据库相关的功能时检查数据完整性和错误。

检查是否正确执行了所有数据库查询，检索了数据并正确更新了数据。有关数据库测试的更多信息可能是数据库上的负载，我们将在下面的Web负载或性能测试中解决此问题。

在测试网站功能时，还应该测试以下内容：

链接

i. 内部链接
ii.外部链接
iii.邮件链接
iv.断开的链接

表格
  1. 字段验证
  2. 错误输入的错误消息
  3. 可选和必填字段
数据库

测试将针对数据库完整性进行。

可用性测试

可用性测试是测量系统的人机交互特性，并确定不足以进行纠正。

导航测试

导航是指用户如何浏览网页、不同的控件（如按钮、框）或用户如何使用页面上的链接浏览不同的页面。

可用性测试应包括以下内容
  • 该网站应易于使用。
  • 所提供的说明应非常清楚。
  • 检查提供的说明是否完全符合其目的。
  • 每个页面上都应提供主菜单。
  • 它应该足够一致。
内容检查

内容应符合逻辑且易于理解。检查拼写错误。深色背景使用会让用户不喜欢，不应在网站主题中使用。应该遵循一些用于网页和内容构建的标准颜色。这些是普遍接受的标准。内容应该是有意义的。所有锚文本链接都应正常工作。图像应以适当的尺寸正确放置。这些是页面开发中应遵循的一些基本重要标准。你的任务是验证所有内容以进行 UI 测试。

用于帮助用户搜索信息

与搜索选项一样，站点地图也有助于处理文件等。站点地图应与网站上的所有链接一起提供，并具有正确的导航树视图。检查站点地图上的所有链接。

“在网站中搜索”选项将帮助用户轻松快速地找到他们正在寻找的内容页面。这些都是可选项目，如果存在，则应对其进行验证。

接口测试

在页面测试中，应测试服务器端接口。这可以通过验证通信是否正确完成。应测试服务器与软件、硬件、网络和数据库的兼容性。

主要接口有：

  • 页面服务器和应用程序服务器接口
  • 应用程序服务器和数据库服务器接口。

检查这些服务器之间的所有交互是否都已执行，错误是否得到正确处理。如果数据库或 Web 服务器为应用程序服务器的任何查询返回错误消息，那么应用程序服务器应捕获这些错误消息并将其适当地显示给用户。

检查如果用户中断其中的任何事务会发生什么情况？检查如果与Web服务器的连接在两者之间重置会发生什么？

兼容性测试

网站的兼容性是一个非常重要的测试方面。查看要执行的兼容性测试：

  • 浏览器兼容性
  • 操作系统兼容性
  • 移动浏览
  • 打印选项
浏览器兼容性：

在我的网络测试生涯中，我经历过这是网站测试中最具影响力的部分。
某些应用程序非常依赖于浏览器。不同的浏览器具有不同的配置和设置，您的网页应与之兼容。

您的网站编码应该是跨浏览器平台兼容的。如果您正在使用 Java 脚本或 AJAX 调用 UI 功能，则执行安全检查或验证会给页面应用程序的浏览器兼容性测试带来更多压力。

在不同的浏览器上测试页面应用程序，如Edge，Firefox，Chrome，Netscape Navigator，AOL，Safari，Opera浏览器和不同版本。

操作系统兼容性：

Web 应用程序中的某些功能是它可能不与所有操作系统兼容。Web开发中使用的所有新技术，如图形设计和界面调用（如不同的API）可能并非在所有操作系统中都可用。

因此，在具有不同操作系统风格的Windows，Unix，MAC，Linux，Solaris等不同操作系统上测试您的Web应用程序。

移动浏览：

在移动浏览器上测试网页。移动设备上也可能存在兼容性问题。

打印选项：

如果要提供页面打印选项，请确保字体，页面对齐方式，页面图形等正确打印。页面应适合纸张大小或根据打印选项中提到的大小。

性能测试

页面 应用程序应承受重负载。页面性能测试应包括：

  • 网络负载测试
  • 网络压力测试

在不同的互联网连接速度下测试应用程序性能。

页面负载测试：需要测试是否有多个用户正在访问或请求同一页面。系统能否维持峰值负载时间？站点应处理许多并发用户请求、来自用户的大量输入数据、同时连接到数据库、特定页面上的重负载等。

网络压力测试：通常，应力意味着将系统拉伸到其指定的限值之外。执行页面压力测试以通过施加压力来破坏站点，并检查系统如何对压力做出反应以及如何从崩溃中恢复。压力通常放在输入字段，登录和注册区域。

在页面性能中，在不同操作系统和不同硬件平台上测试网站功能时，会检查软件和硬件内存泄漏错误。

性能测试可用于了解网站的可伸缩性，或对第三方产品（如服务器和中间件）环境中的性能进行基准测试，以便进行潜在购买。

连接速度

在各种网络上经过测试，如拨号，ISDN等。

页面加载

页面加载性能可以用Lighthouse进行测试。

i.每次最多承受用户数？
ii. 检查峰值负载和系统的行为方式

iii.用户访问的大量数据

压力测试

i. 连续载荷
ii.内存、CPU、文件处理等性能

安全测试

以下是页面安全测试的一些测试用例：

  • 通过将内部 URL 直接粘贴到浏览器地址栏中而不登录来进行测试。不应打开内部页面。
  • 如果您使用用户名和密码登录并浏览内部页面，请尝试直接更改URL选项。即，如果您正在检查一些发布者站点统计信息，则发布者站点 ID = 123。尝试直接将 URL 站点 ID 参数更改为与登录用户无关的其他站点 ID。应拒绝此用户查看其他人的统计信息的访问权限。
  • 尝试在输入字段中输入一些无效的输入，例如登录用户名，密码，输入文本框等。检查系统对所有无效输入的反应。
  • 页面目录和文件不应直接访问，除非为它们提供了下载选项。
  • 测试验证码以自动执行脚本登录。
  • 测试 SSL 是否用于安全措施。如果使用，当用户从不安全的页面HTTP://切换到安全 HTTPS:// 页面时，应显示正确的消息，反之亦然。
  • 所有事务、错误消息和安全漏洞尝试都应记录在服务器上某个位置的日志文件中。

测试页面安全性的主要原因是识别潜在的漏洞并随后修复它们。

  • 网络扫描
  • 漏洞扫描
  • 密码破解
  • 日志审查
  • 完整性检查器
  • 病毒检测
